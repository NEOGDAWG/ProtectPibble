## Contributing

### Branching (hackathon-friendly)

- `main`: stable / demo-ready
- feature branches: `feat/<short-name>` or `fix/<short-name>`

### Pull requests

- Keep PRs small (ideally < 300 lines changed).
- Include a short summary and a basic test plan (how you verified it).

### Local dev

```bash
cp env.example .env
./scripts/bootstrap
./scripts/dev
```
